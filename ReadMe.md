基础相机应用功能与交互文档

1. 应用概述

应用名称: MyCamera
核心功能: 基于Android CameraX的轻量级相机应用，实现基础的拍照、预览、相机控制等功能
技术栈: Android + Kotlin + CameraX + MVVM架构

2. 核心功能模块

2.1 相机预览

• 实时预览: 显示相机捕捉的实时画面

• 方向适配: 支持横竖屏自动适配

• 比例控制: 默认16:9预览比例

• 画面镜像: 前置摄像头自动镜像处理

2.2 拍照功能

• 基础拍照: 单次点击快门按钮拍照

• 照片保存: 自动保存照片到系统相册

• EXIF信息: 保存拍摄时间、位置(需权限)、设备信息

• 质量设置: 可配置照片质量(高/中/低)

2.3 相机控制

• 摄像头切换: 前后置摄像头切换

• 闪光灯控制: 自动/开启/关闭三种模式

• 对焦控制: 点击屏幕指定对焦点

• 缩放控制: 双指捏合缩放

• 曝光补偿: 滑动调整曝光值

2.4 基础交互

• 快门按钮: 圆形按钮，点击拍照

• 摄像头切换按钮: 前置/后置切换

• 闪光灯按钮: 模式循环切换

• 相册入口: 查看最近拍摄的照片

• 设置入口: 进入相机设置界面

4. 功能详细说明

4.1 拍照流程


用户点击快门 → 对焦锁定 → 拍照处理 → 图片保存 → 保存成功提示


详细步骤:
1. 用户点击快门按钮
2. 系统进行自动对焦(如有需要)
3. CameraX捕获图片
4. 图片EXIF信息处理
5. 保存到系统相册指定目录
6. 显示保存成功提示动画

4.2 摄像头切换逻辑

• 默认启动后置摄像头

• 点击切换按钮在前置/后置间切换

• 切换时保持当前设置(闪光灯状态、比例等)

• 切换过程显示过渡动画

4.3 闪光灯控制

三种模式循环:
1. 自动模式(AUTO): 系统根据环境光自动判断
2. 常亮模式(ON): 持续开启补光
3. 关闭模式(OFF): 不开启闪光灯

状态指示:
• 自动模式: ⚡(带A图标)

• 开启模式: ⚡(实心图标)

• 关闭模式: ⚡(灰色图标)

4.4 对焦功能

• 自动对焦: 半按快门或点击屏幕触发

• 对焦区域: 支持多点对焦

• 对焦指示: 显示对焦框及对焦状态

• 对焦锁定: 长按锁定对焦

7.2 权限请求流程

1. 应用启动检查必需权限
2. 权限缺失时弹窗请求
3. 用户拒绝后引导至设置页
4. 权限获取成功后初始化相机

8. 错误处理

8.1 常见错误场景

1. 相机被占用: 提示"相机被其他应用占用"
2. 权限拒绝: 引导用户开启权限
3. 存储空间不足: 提示清理空间
4. 相机初始化失败: 重启应用提示

8.2 异常处理策略

• 网络异常: 本地缓存操作

• 存储异常: 降级到应用私有目录

• 相机异常: 尝试重新初始化

9. 性能优化

9.1 内存管理

• 图片加载使用Glide并配置适当缓存

• 及时释放相机资源

• 避免内存泄漏(使用弱引用)

9.2 响应优化

• 相机操作在后台线程处理

• UI更新在主线程执行

• 预览帧率优化(30fps)
